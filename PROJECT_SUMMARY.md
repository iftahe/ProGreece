# סיכום התקדמות פרויקט ProGreece

מסמך זה מרכז את כל העבודה שבוצעה בפרויקט עד כה, כולל ארכיטקטורה, פיתוח צד-שרת, צד-לקוח, מסדי נתונים וסקריפטים לטעינת נתונים.

## 1. סקירה כללית (Overview)
המערכת היא אפליקציית Web לניהול תזרים מזומנים (Cash Flow) עבור פרויקט "Greece Project". המערכת מאפשרת מעקב אחרי הוצאות והכנסות בפועל (Actual) מול תכנון (Planned), הצגת דוחות גרפיים וניהול תנועות יומיומי.

---

## 2. ארכיטקטורה וטכנולוגיות

### צד שרת (Backend) - Python
*   **Framework**: FastAPI.
*   **Database**: SQLite (קובץ מקומי).
*   **ORM**: SQLAlchemy לניהול המודלים והשאילתות.
*   **API**: חשיפת Endpoints עבור הפרונט (קבלת פרויקטים, חשבונות, תנועות, דוחות ותחזיות).
*   **קובץ ראשי**: `main.py`.

### צד לקוח (Frontend) - React
*   **Build Tool**: Vite.
*   **Styling**: Tailwind CSS.
*   **Charts**: Recharts (להצגת גרפים של תזרים).
*   **HTTP Client**: Axios (ניהול קריאות לשרת).
*   **Routing**: React Router.

---

## 3. מודלים ונתונים (Database Schema)
הוגדרו הטבלאות הבאות ב-`models.py`:
*   **Projects**: ניהול פרויקטים (למשל "Athens Luxury").
*   **Accounts**: ניהול חשבונות (בנק, ספקים, לקוחות, חשבונות מערכת כמו מע"מ).
*   **Transactions**: תנועות כספיות בפועל (תאריך, סכום, מ-חשבון, ל-חשבון, מע"מ).
*   **ProjectPaymentPhase**: תכנון הוצאות לפי שלבים (Cost Planning).
*   **CustomerPaymentPlan**: תכנון הכנסות מלקוחות (Income Planning).

---

## 4. פיצ'רים מרכזיים שפותחו

### א. ייבוא נתונים (Data Import Automation)
פותחו סקריפטים חכמים לטעינת נתונים היסטוריים ותכנוניים:
1.  **`import_real_data.py`**:
    *   קורא קובץ אקסל/CSV של תנועות בנק (היסטוריה).
    *   מזהה אוטומטית עמודות (גם בעברית: 'תאריך', 'סכום', 'מ', 'אל').
    *   מטפל בקידודים שונים ובפורמטים של תאריכים.
    *   מזהה חשבונות מערכת (כגון "מעמ", "עמלה") ויוצר חשבונות חדשים אוטומטית אם אינם קיימים.
    *   ממיר סוגי נתונים (Decimal ל-Float) לתאימות מלאה ל-SQLite.
2.  **`import_plans.py`**:
    *   קורא קובץ `plans.csv` המכיל תכנון הוצאות והכנסות.
    *   מפרמט תאאריכים וסכומים ומכניס לטבלאות התכנון (`ProjectPaymentPhase`, `CustomerPaymentPlan`).
3.  **`init_db.py`**: סקריפט לאתחול הטבלאות ויצירת פרויקט ברירת מחדל.

### ב. דאשבורד (Dashboard)
*   **גרף משולב**: מציג הכנסות והוצאות (עמודות) ויתרה מצטברת (קו) לאורך זמן.
*   **טבלת תזרים**: מציגה את הנתונים הטבלאיים של הגרף.
*   **לוגיקה**: שילוב של נתוני אמת (Transactions) עם נתוני תכנון (Plans) לתצוגת Cash Flow מלאה.

### ג. מסך תנועות (Transactions)
*   **טופס הוספה**: יצירת תנועה חדשה בצורה ידנית (בחירת פרויקט, חשבונות, תאריך וסטטוס).
*   **טבלת היסטוריה**: תצוגה של כל התנועות הקיימות במערכת עם חיווי ויזואלי לסטטוס (Planned/Executed).
*   **עדכון בזמן אמת**: הטבלה מתרעננת מיד לאחר הוספת תנועה.
